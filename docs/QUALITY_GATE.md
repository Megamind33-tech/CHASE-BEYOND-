# Quality Gate

Every feature must be verified before it is called complete.

Required report:

- Typecheck result.
- Test result.
- Build result.
- Browser check result.
- Screenshot or recording when visual work changed.
- Known limitations.

Do not accept a change that leaves the app blank, creates controls that do nothing, adds packages without a decision note, or removes checks only to pass.

For the first foundation, prove that the app opens, the Babylon scene renders, a video source appears on a 3D screen mesh, sources can be replaced, sets can be replaced, OBS can capture the output, and a 30-minute stability run is documented.
