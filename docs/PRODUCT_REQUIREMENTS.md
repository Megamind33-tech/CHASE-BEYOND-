# Product Requirements

## Product name

Beyond.

## Product definition

Beyond is a lightweight virtual broadcast studio for operators who need to load a 3D studio set, place real media sources inside it, switch sources, switch sets, and output through OBS without technical confusion.

## Primary user

The first user is a small broadcast operator, online TV operator, church/media team, music show producer, or studio technician who wants professional-looking virtual studio output without heavy virtual production hardware.

## First product promise

Beyond must make this workflow possible:

1. Open the app.
2. Choose a virtual set.
3. Choose a media source.
4. See the media source inside the 3D set.
5. Replace the source.
6. Replace the set.
7. Capture the output in OBS.

## What Beyond is not yet

Beyond is not yet:

- A full OBS replacement.
- A full vMix replacement.
- A full Zero Density replacement.
- A custom encoder.
- A hardware capture-card system.
- An AI set generator.
- A cloud collaboration platform.

## Product principles

- Working simple beats impressive broken.
- Real controls beat fake dashboard beauty.
- Stability beats feature count.
- Operators must see clear errors.
- Every important claim must be checked.

## Launchable foundation

The first launchable foundation is not feature-rich. It is credible only if it opens, renders a 3D studio, displays a real media source inside the studio, switches source, switches set, and can be captured by OBS.

## Operator experience target

The app should feel like a focused broadcast tool, not a generic admin dashboard. The operator must always know:

- Which set is active.
- Which source is active.
- Whether the source is working.
- Whether the 3D runtime is healthy.
- What went wrong if something fails.
